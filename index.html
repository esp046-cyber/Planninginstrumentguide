1 Commissioning Phases Overview

Pre-Commissioning Phase:

  • Definition: Activities before introducing process fluids/energy
  • Duration: 3-6 months for mega refinery
  • Key Activities: Installation verification, loop checks, continuity testing
  • Completion Criteria: All systems energized and ready for process introduction

Commissioning Phase:

  • Definition: Testing with process fluids to verify performance
  • Duration: 4-8 months
  • Key Activities: Functional testing, calibration verification, integration testing
  • Completion Criteria: All systems meet design specifications

Startup Phase:

  • Definition: Initial operation to stable production
  • Duration: 2-4 months
  • Key Activities: Performance optimization, operator training, troubleshooting
  • Completion Criteria: Plant achieves design capacity at specifications
Planning Rule: Allocate 40% of commissioning effort to pre-commissioning (prevention), 40% to commissioning (verification), 20% to startup support (optimization).

2 Pre-Commissioning Activities

A. Installation Verification (Punch List Clearance):

B. Loop Checks (End-to-End Verification):

  • Purpose: Verify complete signal path from sensor to DCS to final element
  • Method: Simulate input at field device, verify response at control room and actuator
  • Documentation: Loop check sheets with as-found/as-left values
  • Typical Sequence:
    • Apply calibrated input signal (0%, 25%, 50%, 75%, 100%)
    • Verify DCS reading matches within Â±0.5% accuracy
    • Test alarms at setpoints (confirm annunciation)
    • Verify control output to valve/actuator
    • Test manual/auto mode switching
    • Verify shutdown/interlock logic if applicable
Critical: Loop checks must be 100% complete before introducing hydrocarbons. Incomplete loops = safety hazard and startup delays.

C. Power-Up and Energization:

  • Verify power supply voltage and phase rotation
  • Check grounding resistance (<1 ohm for instrument ground)
  • Energize UPS systems and verify switchover
  • Power up DCS cabinets sequentially per vendor procedure
  • Load DCS configuration and verify database integrity
  • Test emergency shutdown systems (E-stop, ESD)

3 Systematic Completion Strategy

Completion Hierarchy:

Level 1 - Tag Completion:

  • Individual instrument fully installed, tested, and functional
  • Certificate: Instrument Completion Certificate (ICC)
  • Signoff: Instrument Technician + QC Inspector

Level 2 - Loop Completion:

  • Complete control loop tested end-to-end
  • Certificate: Loop Completion Certificate (LCC)
  • Signoff: Commissioning Engineer + Operations Representative

Level 3 - System Completion:

  • All loops for a subsystem (e.g., FCC reactor control)
  • Certificate: Subsystem Completion Certificate (SCC)
  • Signoff: Commissioning Manager + Area Supervisor

Level 4 - Mechanical Completion (MC):

  • Entire process unit ready for hydrocarbon introduction
  • Certificate: Mechanical Completion Certificate (MCC)
  • Signoff: Project Manager + Client Representative
  • Prerequisite for PSSR (Pre-Startup Safety Review)
Planning Tip: Track completion percentages weekly. Aim for 95%+ Level 1 before starting Level 2. Allows parallel progression without rework.

4 Commissioning Test Procedures

Instrument Calibration Verification:

  • Frequency: Every instrument before commissioning
  • Method: Compare to certified reference standard
  • Acceptance: Within manufacturer's stated accuracy (typically Â±0.5%)
  • Documentation: Calibration certificate with serial numbers, test data, technician signature

Functional Testing Examples:

1. Level Control Loop:

  • Fill vessel to verify transmitter range (0-100%)
  • Test level alarms: LAL, LAH, LALL, LAHH
  • Verify control valve response to level changes
  • Test automatic level control in various ranges
  • Verify emergency shutdown at LALL/LAHH

2. Temperature Control Loop:

  • Simulate temperature using calibrator at thermowell
  • Verify DCS indication and trend
  • Test cascade control (if applicable)
  • Verify heater/cooler control valve response
  • Test high temperature trip function

3. Analyzer System:

  • Calibrate with certified gas standards (zero, span, mid-point)
  • Verify sample conditioning system (pressure, temperature, flow)
  • Test analyzer alarms and auto-validation
  • Verify data transmission to DCS
  • Document response time and linearity
Safety First: All functional tests requiring process simulation must have approved test procedures, JSA (Job Safety Analysis), and permit to work.

5 DCS Commissioning

Site Acceptance Test (SAT) Components:

Control Logic Verification:

  • PID Control: Tuning parameters, auto/manual bumpless transfer
  • Cascade Control: Master/slave relationship, mode handling
  • Sequential Control: Step logic, permissives, interlocks
  • Batch Control: Recipe execution, phase transitions
  • Advanced Control: MPC, fuzzy logic, custom algorithms
Simulation Testing: Use DCS simulator mode to test complex control strategies before connecting to field instruments. Reduces risk during actual commissioning.

9 Punch List Management

Punch List Categories:

Category A (Safety Critical):

  • Must be complete before unit startup
  • Examples: SIS loops, fire & gas, PSVs, emergency isolation
  • Target: 0 Category A items at PSSR

Category B (Operationally Critical):

  • Required for normal operation but not immediate safety risk
  • Examples: Level controls, flow measurements, analyzers
  • Target: <5% Category B open at startup

Category C (Non-Critical):

  • Cosmetic or convenience items
  • Examples: Missing labels, paint touchup, spare parts delivery
  • Acceptable: Complete within 30 days post-startup

Punch List Tracking:

  • Database: Use electronic system (e.g., Meridian, Completr, custom Excel)
  • Fields: Item #, Description, Location, Category, Responsible party, Target date, Status
  • Reporting: Daily during peak commissioning, weekly thereafter
  • Aging Report: Flag items >7 days overdue for escalation
Punch List Discipline: Do not allow "punch list culture" where poor installation is accepted. Require contractors to complete work properly the first time.

10 Performance Testing

Instrument Performance Metrics:

1. Accuracy:

  • Measured value vs. true value (reference standard)
  • Typical: Â±0.5% of span for process transmitters
  • Test: 5-point calibration (0, 25, 50, 75, 100%)

2. Repeatability:

  • Consistency of readings under same conditions
  • Typical: Â±0.1% of span
  • Test: 10 repeated measurements at same setpoint

3. Response Time:

  • Time to reach 63.2% of final value (time constant)
  • Critical for control loops and SIS
  • Test: Step change input, measure time to stabilize

Control Loop Performance:

  • Setpoint Tracking: PV follows SP within Â±2% in steady state
  • Load Rejection: Returns to setpoint within specified time after disturbance
  • Overshoot: <10% for most process control loops
  • Stability: No sustained oscillations or limit cycling
Performance Guarantee: DCS vendor typically guarantees <2% steady-state error and <5 second response time for standard control loops.

12 Lessons Learned & Continuous Improvement

Common Commissioning Challenges:

  • Late Design Changes: P&ID revisions during commissioning require rework
  • Incomplete Construction: Trying to commission before installation complete
  • Inadequate Testing: Skipping steps to meet schedule, problems found at startup
  • Documentation Lag: Paperwork not keeping pace with physical work
  • Resource Constraints: Insufficient technicians or test equipment
  • Vendor Delays: Support personnel unavailable when needed

Best Practices:

  • Freeze design at 90% - resist late changes
  • Don't start commissioning until construction >95% complete in area
  • Use pre-commissioning checklists to verify readiness
  • Real-time documentation - complete paperwork same day as test
  • Secure vendor commitments 6 months in advance
  • Maintain 20% equipment spares for breakdowns
  • Daily coordination meetings during peak commissioning

Post-Commissioning Review:

  • Schedule within 30 days of plant handover
  • Include commissioning team, operations, maintenance
  • Review schedule performance (planned vs. actual)
  • Analyze major issues and root causes
  • Document recommendations for future projects
  • Update commissioning procedures and templates
Continuous Improvement: Build a commissioning knowledge base. Each project should improve on the last through captured lessons learned.

3 Monte Carlo Simulation

Quantitative risk analysis using probability distributions:

Why Use Monte Carlo: Single-point duration estimates ignore uncertainty. Monte Carlo provides probability-based completion dates.

Process:

  • Step 1: Identify high-risk activities (long-lead procurement, FAT, etc.)
  • Step 2: Assign three-point estimates (Optimistic, Most Likely, Pessimistic)
  • Step 3: Run 1000+ simulation iterations
  • Step 4: Analyze probability distribution of completion dates

Example Application:

  • DCS Delivery: Optimistic = 20 weeks, Likely = 24 weeks, Pessimistic = 32 weeks
  • Analyzer Delivery: Optimistic = 16 weeks, Likely = 20 weeks, Pessimistic = 28 weeks

4 Critical Chain Project Management (CCPM)

Focus on resource dependencies, not just task dependencies:

  • Remove Task Padding: Eliminate safety buffers in individual tasks
  • Add Project Buffer: Consolidate buffer at end of critical chain
  • Feeding Buffers: Protect critical chain from non-critical delays
  • Resource Buffers: Ensure critical resources are available when needed

CCPM vs Traditional CPM:

  • Traditional: Cable pulling = 15 days (includes 3-day buffer)
  • CCPM: Cable pulling = 12 days + feeding buffer protects next task
  • Result: Earlier starts, more realistic durations, better resource utilization
Buffer Management: Monitor buffer consumption rate. If >50% buffer consumed at <50% task progress, initiate recovery actions.

1 Contract Types in Oil & Gas

A. Lump Sum Turnkey (LSTK):

  • Definition: Fixed price for defined scope, contractor assumes all risk
  • When Used: Well-defined scope, minimal changes expected
  • Instrumentation Impact: Design must be 90%+ complete before contract award
  • Risk: Scope changes very expensive (20-30% markup)
  • Planning Focus: Freeze design early, strict change control

B. Engineering, Procurement, Construction (EPC):

  • Definition: Single contractor for full project delivery
  • Variations: Can be lump sum, cost-reimbursable, or hybrid
  • Instrumentation Scope: Complete responsibility from design through commissioning
  • Planning Role: Coordinate all phases, single point accountability

C. EPCM (Engineering, Procurement, Construction Management):

  • Definition: EPCM contractor manages, client holds construction contracts
  • Payment: Cost-reimbursable with fee (hourly rates + markup)
  • Instrumentation Impact: More client involvement, flexible scope changes
  • Planning Focus: Coordinate multiple construction contractors

D. Cost-Reimbursable (Cost-Plus):

  • Definition: Client pays actual costs + contractor fee
  • Fee Types: Fixed fee, percentage fee, target cost with incentives
  • When Used: Scope uncertainty, fast-track projects, emergencies
  • Risk: Cost overruns absorbed by client
  • Planning Focus: Detailed cost tracking, productivity monitoring
Contract Selection Impact: LSTK requires most detailed upfront planning. EPCM offers most flexibility for instrumentation changes during execution.

  • Subcontractor performance issues (vendor risk)
  • Normal weather (within statistical norms)
Claim Prevention: Document all changes in real-time. Unsigned "verbal changes" lead to claims. Use formal VO process even for small changes.

8 Warranty Provisions

Standard Warranty Terms:

  • Duration: 12-24 months from substantial completion or startup (whichever comes first)
  • Coverage: Defects in materials, workmanship, and design
  • Remedy: Repair or replace at vendor's expense
  • Response Time: Critical: 24 hrs, Non-critical: 7 days

Warranty Exclusions (typical):

  • Improper operation or maintenance by client
  • Modifications by parties other than vendor
  • Normal wear and tear (consumables, gaskets)
  • Damage from external causes (lightning, flooding)
  • Use beyond design specifications

Extended Warranty Considerations:

  • Cost: 3-5% of equipment value per additional year
  • Benefit: Predictable costs, vendor expertise available
  • Analysis: Compare to in-house maintenance capability
  • DCS Systems: Often worth extending to 3-5 years due to complexity
Planning Task: Create warranty tracking database. Schedule warranty expiration reviews 90 days before end to identify any latent defects before warranty expires.

9 Intellectual Property (IP) Rights

IP Ownership in Instrumentation Contracts:

Client-Owned IP:

  • Project-specific engineering deliverables (P&IDs, loop diagrams, specs)
  • DCS configuration databases (graphics, control logic, alarms)
  • Custom software developed for the project
  • As-built documentation and operating procedures

Vendor-Owned IP:

  • Proprietary software (DCS operating system, engineering tools)
  • Standard equipment designs and algorithms
  • Pre-existing vendor intellectual property
  • Vendor's methodology and know-how

License Terms:

  • Runtime Licenses: Perpetual for installed systems, or subscription-based
  • Engineering Licenses: Often limited term, may require renewal
  • Expansion Rights: Can client add I/O without re-licensing?
  • Backup/DR Rights: Can client restore on new hardware in emergency?
Critical: Ensure DCS license agreement permits client to restore system from backups without vendor involvement. Essential for disaster recovery.

10 Dispute Resolution Mechanisms

Escalation Hierarchy (typical):

Level 1 - Project Management:

  • Instrument Engineer & Vendor Project Manager attempt resolution
  • Timeframe: 7 days
  • Most common issues: Technical disputes, minor delays

Level 2 - Senior Management:

  • Project Director & Vendor VP meet to resolve
  • Timeframe: 14 days from Level 1 failure
  • Issues: Cost claims, schedule extensions, scope interpretations

Level 3 - Mediation:

  • Neutral third-party mediator facilitates agreement
  • Non-binding, collaborative process
  • Timeframe: 30-60 days
  • Success Rate: ~70% of commercial disputes

Level 4 - Arbitration:

  • Binding decision by arbitrator(s)
  • Governed by rules (ICC, LCIA, SIAC, AAA)
  • Timeframe: 6-18 months
  • Preferred over litigation: Faster, confidential, enforceable internationally

Level 5 - Litigation:

  • Court proceedings (last resort)
  • Public, lengthy (2-5 years), expensive
  • Rarely used in international oil & gas contracts
Best Practice: Resolve disputes at lowest level possible. Document all agreements in writing (meeting minutes, email confirmations) to prevent future claims.

11 Incoterms & Delivery Terms

Common Incoterms for Instrumentation:

EXW (Ex Works):

  • Vendor responsibility ends at factory gate
  • Client arranges all transportation, insurance, customs
  • Lowest vendor price, highest client logistics burden

FOB (Free on Board):

  • Vendor delivers to port and loads on vessel
  • Risk transfers when goods cross ship's rail
  • Client pays ocean freight and insurance

CIF (Cost, Insurance, Freight):

  • Vendor pays freight and insurance to destination port
  • Risk still transfers at origin port loading
  • Client handles customs clearance and inland transport

DDP (Delivered Duty Paid):

  • Vendor responsible for all costs to final destination
  • Includes freight, insurance, customs, duties, inland transport
  • Highest vendor price, lowest client hassle
  • Preferred for complex equipment (analyzers, DCS) from international vendors
Planning Recommendation: For critical long-lead items (DCS, analyzers), use DDP to site. Vendor has better logistics expertise and can expedite if needed.

12 Insurance & Indemnity

Required Insurance Coverage:

1. Marine Cargo Insurance:

  • Coverage: Loss or damage during international shipment
  • Amount: 110% of CIF value (covers profit margin)
  • Terms: Warehouse to warehouse, all risks

2. Professional Indemnity (PI):

  • Coverage: Errors in engineering design and specifications
  • Amount: $5-10M minimum for mega refinery projects
  • Who: Engineering contractors and design consultants

3. Public Liability:

  • Coverage: Third-party injury or property damage
  • Amount: $10-20M per occurrence
  • Who: All contractors working on site

4. Employer's Liability / Workers Compensation:

  • Coverage: Employee injury or death
  • Amount: Per local regulations (often unlimited)
  • Who: All parties with employees on site

Indemnification Clauses:

  • Mutual Indemnity: Each party indemnifies for its own negligence
  • Consequential Damages: Often excluded (loss of production, profit)
  • Liability Cap: Typically 100% of contract value for general claims
  • Unlimited Liability: Gross negligence, willful misconduct, IP infringement
Planning Impact: Verify all vendors maintain required insurance. Request certificates of insurance before equipment delivery. Uninsured vendors = project risk.
